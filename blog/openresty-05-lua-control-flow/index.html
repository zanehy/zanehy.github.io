05 | Lua 流程控制与函数

发表于： 2026-03-08 分类于： OpenResty
字数： 2606 阅读：≈ 6分钟 浏览：

上一篇我们把 Lua 的六种数据类型摸了个遍。有了数据，下一步自然是：怎么用代码控制这些数据的流向？

这篇文章讲 Lua 的流程控制和函数系统。内容不多，但有几个地方和主流语言不太一样，如果不注意，写出来的代码虽然能跑，但很可能埋着隐患。

if / elseif / else：条件判断

Lua 的 if 语法和大多数语言差不多，但有两点值得注意：用 then 和 end 包围，不用花括号。

local score = 85

if score >= 90 then
    print("优秀")
elseif score >= 60 then
    print("及格")
else
    print("不及格")
end

多条件分支用 elseif（注意是一个词，不是 else if）。

在 OpenResty 里，条件判断随处可见，比如根据请求方法分流处理：

local method = ngx.req.get_method()

if method == "GET" then
    handle_get()
elseif method == "POST" then
    ngx.req.read_body()
    handle_post()
else
    ngx.exit(405)   -- Method Not Allowed
end

回忆上一篇说的：只有 nil 和 false 是假。所以判断一个值是否存在，直接 if val then 就够了，不需要写 if val ~= nil then（虽然两者等价）：

local token = ngx.req.get_headers()["Authorization"]
if token then
    -- token 存在，继续处理
else
    ngx.exit(401)
end

for 循环：两种完全不同的写法

Lua 的 for 有两种形式，用途完全不同，不要混用。

数值 for（numeric for）

适合按数字步长遍历：

-- 从 1 到 5，步长 1
for i = 1, 5 do
    print(i)    -- 1 2 3 4 5
end

-- 从 10 到 1，步长 -2（倒序）
for i = 10, 1, -2 do
    print(i)    -- 10 8 6 4 2
end

格式是 for 变量 = 起始值, 终止值, 步长 do，步长默认为 1，可以省略。

泛型 for（generic for）

适合遍历 table，配合迭代器使用：

-- ipairs：遍历数组，按顺序，遇到 nil 停止
local fruits = {"apple", "banana", "cherry"}
for i, v in ipairs(fruits) do
    print(i, v)
end
-- 输出：
-- 1  apple
-- 2  banana
-- 3  cherry

-- pairs：遍历字典，无序
local headers = {["Content-Type"] = "application/json", ["X-Token"] = "abc123"}
for k, v in pairs(headers) do
    print(k, v)
end

关键区别：

  • ipairs：从下标 1 开始，遇到第一个 nil 就停止，适合数组
  • pairs：遍历所有 key（包括非数字 key），顺序不确定，适合字典

在 OpenResty 里，遍历请求参数经常用 pairs：

local args = ngx.req.get_uri_args()
for key, val in pairs(args) do
    ngx.log(ngx.INFO, "param: ", key, " = ", val)
end

while 和 repeat：条件循环

while 先判断再执行，repeat 先执行再判断（类似其他语言的 do-while）：

-- while：条件为真时循环
local i = 1
while i <= 5 do
    print(i)
    i = i + 1
end

-- repeat...until：先执行，直到条件为真时退出
local j = 1
repeat
    print(j)
    j = j + 1
until j > 5

在 OpenResty 里，while 常用于处理分页或重试逻辑：

local max_retry = 3
local retry = 0
local ok, err

while retry < max_retry do
    ok, err = connect_to_upstream()
    if ok then break end
    retry = retry + 1
    ngx.log(ngx.WARN, "retry ", retry, ": ", err)
end

if not ok then
    ngx.exit(502)
end

break 和 goto

Lua 支持 break 跳出循环，也支持 goto（Lua 5.2+）跳转到标签：

for i = 1, 10 do
    if i == 5 then
        break   -- 跳出循环
    end
    print(i)
end
-- 输出 1 2 3 4

-- goto 用于跳过某次迭代（Lua 没有 continue，用 goto 模拟）
for i = 1, 5 do
    if i == 3 then
        goto continue   -- 跳过 i==3 的情况
    end
    print(i)
    ::continue::        -- 标签定义
end
-- 输出 1 2 4 5

goto 最常见的用途就是模拟 continue，因为 Lua 没有内置的 continue 关键字。

函数：Lua 的核心构建块

Lua 的函数系统比看起来要丰富得多。先从基础讲起。

定义和调用 

-- 标准写法
local function greet(name)
    return "Hello, " .. name
end

-- 等价写法（函数是值，可以赋给变量）
local greet = function(name)
    return "Hello, " .. name
end

print(greet("OpenResty"))   -- Hello, OpenResty

函数定义时加 local 是好习惯——它将函数限定在当前作用域，避免污染全局环境。在 OpenResty 里，未加 local 的全局变量会带来性能问题（每次访问都要查全局表），这一点后面章节会详细讲。

多返回值

这是 Lua 最有特色的功能之一。一个函数可以返回多个值：

local function parse_header(header)
    local name, value = header:match("^([^:]+):%s*(.+)$")
    return name, value
end

local name, value = parse_header("Content-Type: application/json")
print(name)     -- Content-Type
print(value)    -- application/json

多返回值在 OpenResty 的 API 设计里无处不在。几乎所有可能出错的操作都遵循这个约定：第一个返回值是结果，第二个返回值是错误信息：

local res, err = red:get("user:123")
if not res then
    -- res 是 nil，err 是错误描述字符串
    ngx.log(ngx.ERR, "get failed: ", err)
    return ngx.exit(500)
end

接收多返回值时，多余的值被丢弃，不够的用 nil 补齐：

local function three_values()
    return 1, 2, 3
end

local a, b = three_values()     -- a=1, b=2，3 被丢弃
local x, y, z, w = three_values()  -- x=1, y=2, z=3, w=nil

可变参数

函数参数列表里用 ... 表示可变参数：

local function sum(...)
    local args = {...}      -- 把可变参数打包成 table
    local total = 0
    for _, v in ipairs(args) do
        total = total + v
    end
    return total
end

print(sum(1, 2, 3, 4, 5))   -- 15

select 函数配合可变参数很有用：

local function log(level, ...)
    local n = select("#", ...)      -- 获取参数个数
    local msg = table.concat({...}, " ")    -- 拼接所有参数
    ngx.log(level, msg)
end

log(ngx.INFO, "user", "123", "logged in")

在 OpenResty 里，封装日志函数时经常用到可变参数。

闭包：函数记住了它出生时的环境

闭包是 Lua 里一个强大但容易被忽视的特性。简单说：闭包是一个函数，它能访问定义它时所在作用域的变量，即使那个作用域已经结束。

用一个例子来理解：

local function make_counter()
    local count = 0         -- 局部变量
    return function()       -- 返回一个匿名函数
        count = count + 1   -- 内层函数访问外层的 count
        return count
    end
end

local counter = make_counter()
print(counter())    -- 1
print(counter())    -- 2
print(counter())    -- 3

每次调用 counter()，它都"记住"了上一次的 count 值。这个 count 变量就是闭包捕获的"自由变量"。

闭包在 OpenResty 里的实际用途

1. 封装私有状态（模块模式）：

-- 限流器：每个 IP 一个独立计数器
local function make_limiter(max_count)
    local count = 0
    return {
        check = function()
            count = count + 1
            return count <= max_count
        end,
        reset = function()
            count = 0
        end
    }
end

local limiter = make_limiter(100)
if not limiter.check() then
    ngx.exit(429)   -- Too Many Requests
end

2. 延迟计算（惰性初始化）：

local _config = nil

local function get_config()
    if not _config then
        -- 只在第一次调用时从文件或 Redis 加载配置
        _config = load_config_from_redis()
    end
    return _config
end

3. 回调函数：

local function retry(fn, max_times)
    for i = 1, max_times do
        local ok, err = fn()
        if ok then return true end
        ngx.log(ngx.WARN, "attempt ", i, " failed: ", err)
    end
    return false
end

-- 使用时，fn 是一个闭包，捕获了 host 和 port
retry(function()
    return connect(host, port)
end, 3)

作用域：local 很重要

Lua 的变量有两种：局部变量（local）和全局变量。

x = 10          -- 全局变量，整个程序都能访问
local y = 20    -- 局部变量，只在当前 block 内有效

do
    local z = 30    -- z 只在这个 do...end 块内有效
    print(z)        -- 30
end
print(z)            -- nil，z 已经超出作用域

在 OpenResty 里，强烈建议所有变量都加 local。 原因有两个：

  1. 性能：访问局部变量比全局变量快，因为局部变量存在栈上，全局变量需要查哈希表
  2. 安全：避免不同请求之间的变量污染（全局变量在 worker 进程级别共享）

来看一个危险的例子：

-- 危险！不加 local 的变量是全局的
function handle_request()
    user_id = ngx.var.arg_uid   -- 全局变量！
    -- ... 处理逻辑
end

如果两个请求几乎同时到达，第二个请求可能会覆盖第一个请求的 user_id，导致数据串。加上 local 就安全了：

function handle_request()
    local user_id = ngx.var.arg_uid    -- 局部变量，每个请求独立
    -- ...
end

小结

这篇文章涵盖了 Lua 流程控制和函数系统的核心内容：

  • if/elseif/else：注意只有 nil 和 false 为假，其他都是真
  • for 两种形式：数值 for 按步长遍历，泛型 for 配合 ipairs/pairs 遍历 table
  • while / repeat：先判断 vs 先执行；没有 continue，用 goto 模拟
  • 多返回值：Lua 最有特色的设计，OpenResty API 普遍使用"结果, 错误"双返回值约定
  • 闭包：函数捕获外部变量，适合封装状态、实现回调
  • local：始终加 local，性能和安全两手抓
