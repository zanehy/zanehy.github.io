03 | 高频语法精讲（上）——变量、类型与控制流

发表于： 2026-04-18 分类于： Rust
字数： 3311 阅读：≈ 7分钟 浏览：

有了前两篇的认知框架和工具链基础，现在正式进入 Rust 的语法世界。

这一篇不打算把所有语法过一遍——那样做没有意义，查文档比看文章快得多。这里重点讲那些用得最频繁、最容易和 Go/Python 产生认知冲突的部分。

变量绑定：默认不可变

Rust 里声明变量用 let，但和 Go/Python 最大的不同是：变量默认是不可变的（immutable）。

fn main() {
    let x = 5;
    x = 6; // 编译错误：cannot assign twice to immutable variable
}

要让变量可变，需要显式加 mut：

fn main() {
    let mut x = 5;
    x = 6;
    println!("{}", x); // 6
}

这个设计是刻意的。Rust 鼓励你默认写不可变变量，需要改变时再加 mut——这样一眼就能看出哪些变量会被修改，代码意图更清晰，也减少了意外修改的风险。

和 Go 对比：Go 里 var x int = 5 默认可变，想不可变要用常量 const。Rust 反过来，默认不可变，想可变加 mut。

和 Python 对比：Python 里变量随时可以重新绑定，没有"不可变变量"的概念。Rust 的不可变是绑定层面的——这个名字绑定的值不能变。

Shadowing：同名重新绑定

Rust 允许用同名的 let 遮蔽（shadow）之前的变量：

fn main() {
    let x = 5;
    let x = x + 1;     // shadowing：新的 x 遮蔽了旧的 x
    let x = x * 2;
    println!("{}", x); // 12
}

shadowing 和 mut 的关键区别是：shadowing 允许改变类型，而 mut 不允许。

fn main() {
    let spaces = "   ";           // &str 类型
    let spaces = spaces.len();    // usize 类型，类型变了！
    println!("{}", spaces);       // 3
}

如果用 mut 尝试改变类型：

let mut spaces = "   ";
spaces = spaces.len(); // 编译错误：类型不匹配

shadowing 的典型用途就是对同一个值做一系列变换，但不想起多个名字（spaces_str、spaces_len 这种）。

基础类型

Rust 是静态强类型语言，所有类型在编译期确定。

整数类型：

Rust 提供了从 8 位到 128 位的有符号（i）和无符号（u）整数，以及平台相关的 isize / usize。i32 是默认的整数类型，usize 常用于索引和集合长度。

fn main() {
    let a: i32 = -42;
    let b: u64 = 1_000_000; // 下划线分隔，提高可读性
    let c = 0xff_u8;        // 十六进制，后缀指定类型
    let d = 0b1111_0000;    // 二进制
    let e = 255usize;       // 用于索引的典型类型
    println!("{} {} {} {} {}", a, b, c, d, e);
}

浮点：

fn main() {
    let x: f64 = 3.14;   // 默认浮点类型，64 位
    let y: f32 = 2.0f32; // 后缀指定 32 位
    println!("{} {}", x, y);
}

布尔：

let t = true;
let f: bool = false;
let is_valid = t && !f; // true

字符：

Rust 的 char 是 Unicode 标量值，占 4 字节，能表示任何 Unicode 字符——这和很多语言里字符等于字节不一样：

fn main() {
    let c = 'z';
    let chinese = '中';
    let emoji = '😊';  // 合法的 char
    println!("{} {} {}", c, chinese, emoji);
}

复合类型

Tuple：固定长度的异质集合 

fn main() {
    let tup: (i32, f64, bool) = (500, 6.4, true);

    // 解构
    let (x, y, z) = tup;
    println!("{} {} {}", x, y, z);

    // 下标访问
    println!("{}", tup.0); // 500
    println!("{}", tup.1); // 6.4
}

tuple 适合从函数返回多个值，但元素超过 3 个时可读性变差，建议改用 struct。

Struct：命名字段的结构体 

struct User {
    name: String,
    age: u32,
    active: bool,
}

fn main() {
    let user1 = User {
        name: String::from("Alice"),
        age: 30,
        active: true,
    };

    println!("{} is {}", user1.name, user1.age);

    // 结构体更新语法：只改部分字段，其余从已有实例复制
    let user2 = User {
        name: String::from("Bob"),
        ..user1  // 其余字段从 user1 复制
    };
    println!("{} is {}", user2.name, user2.age); // Bob is 30
}

Enum：枚举，Rust 最强大的类型之一

Rust 的枚举不只是一组命名常量，每个变体可以携带不同类型和数量的数据：

enum Message {
    Quit,                        // 没有数据
    Move { x: i32, y: i32 },    // 带命名字段
    Write(String),               // 带一个 String
    Color(i32, i32, i32),        // 带三个 i32
}

fn process(msg: Message) {
    match msg {
        Message::Quit => println!("quit"),
        Message::Move { x, y } => println!("move to ({}, {})", x, y),
        Message::Write(text) => println!("write: {}", text),
        Message::Color(r, g, b) => println!("color: {} {} {}", r, g, b),
    }
}

fn main() {
    process(Message::Move { x: 10, y: 20 });
    process(Message::Write(String::from("hello")));
}

这种设计在其他语言里通常需要继承或联合类型，Rust 用枚举就能做到，且完全类型安全。标准库里的 Option<T> 和 Result<T, E> 都是枚举，这也是为什么它们能用 match 来处理。

函数：表达式 vs 语句

这是 Rust 和 Go/Python 差异最明显的地方之一。

Rust 区分表达式（expression）和语句（statement）：

  • 语句：执行操作，不返回值，以分号 ; 结尾
  • 表达式：计算并返回一个值，不加分号
fn main() {
    let y = {
        let x = 3;
        x + 1   // 没有分号，这是表达式，值为 4
    };           // 整个块是表达式，y = 4

    println!("{}", y); // 4
}

函数的返回值就是最后一个表达式：

fn add(a: i32, b: i32) -> i32 {
    a + b  // 没有分号，作为返回值
}

fn add_explicit(a: i32, b: i32) -> i32 {
    return a + b; // 用 return 显式返回
}

加了分号就变成语句，返回 ()（unit 类型）：

fn broken_add(a: i32, b: i32) -> i32 {
    a + b; // 加了分号！变成语句，函数实际返回 ()
}          // 编译错误：expected i32, found ()

这个"忘记去掉分号"的错误初学时非常常见，编译器的提示很清晰，顺着看就能定位。

if 也是表达式：

fn main() {
    let condition = true;
    // if 整体是一个表达式，可以直接赋值
    let number = if condition { 5 } else { 6 };
    println!("{}", number); // 5
}

注意两个分支必须是同一类型，否则编译报错。这在 Go 里做不到，因为 Go 的 if 是语句。

match：强大的模式匹配

match 是 Rust 最强大的控制流结构，远比其他语言的 switch 强大。

基本用法：

fn main() {
    let number = 7;

    match number {
        1 => println!("one"),
        2 | 3 | 5 | 7 => println!("prime"),  // | 匹配多个值
        13..=19 => println!("a teen"),         // ..= 闭区间范围
        _ => println!("something else"),       // _ 通配符，必须穷尽
    }
}

match 的几个关键特性：

  • 必须穷尽所有可能，不能遗漏——这是编译期强制检查的
  • 每个分支叫 arm，格式是 模式 => 表达式
  • match 本身也是表达式，可以赋值给变量

匹配枚举并解构：

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(String), // 携带州名
}

fn value_in_cents(coin: Coin) -> u32 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter(state) => {
            println!("Quarter from {}", state);
            25
        }
    }
}

fn main() {
    let q = Coin::Quarter(String::from("Alaska"));
    println!("{} cents", value_in_cents(q));
}

match guard：在模式上附加额外条件：

fn describe_number(n: i32) -> &'static str {
    match n {
        0 => "zero",
        n if n < 0 => "negative",   // match guard
        1..=9 => "single digit",
        _ => "large",
    }
}

fn main() {
    println!("{}", describe_number(-5));  // negative
    println!("{}", describe_number(7));   // single digit
    println!("{}", describe_number(100)); // large
}

if let：简化单分支匹配

match 要求穷尽所有情况，但有时候你只关心一种情况，其他都忽略。用 if let 更简洁：

fn main() {
    let some_value: Option<i32> = Some(42);

    // 用 match——必须写出 None 分支
    match some_value {
        Some(v) => println!("got {}", v),
        None => {}
    }

    // 用 if let——只处理关心的情况
    if let Some(v) = some_value {
        println!("got {}", v);
    }

    // if let 可以加 else
    if let Some(v) = some_value {
        println!("got {}", v);
    } else {
        println!("nothing");
    }
}

if let 是 match 的语法糖，选哪个看代码的清晰程度。如果只有一个分支需要处理，if let 更简洁；如果需要处理多种情况，用 match。

while let：循环直到模式不匹配 

fn main() {
    let mut stack = vec![1, 2, 3];

    // 不断 pop，直到 stack 为空（pop 返回 None）
    while let Some(top) = stack.pop() {
        println!("{}", top); // 依次打印 3, 2, 1
    }
}

常用但易混的语法

.. 的两种用法

用于范围匹配（..= 是闭区间）：

fn main() {
    let x = 5;
    match x {
        1..=5 => println!("1 到 5"),
        6..=10 => println!("6 到 10"),
        _ => println!("其他"),
    }
}

用于忽略结构体的剩余字段：

struct Point {
    x: i32,
    y: i32,
    z: i32,
}

fn main() {
    let p = Point { x: 1, y: 2, z: 3 };

    // 只取 x，忽略其他字段
    let Point { x, .. } = p;
    println!("x = {}", x);

    // 在 match 中忽略 tuple 的部分元素
    let numbers = (2, 4, 8, 16, 32);
    match numbers {
        (first, .., last) => println!("first={}, last={}", first, last),
    }
}

_ 通配符 

fn main() {
    // 解构时忽略某个值
    let (x, _, z) = (1, 2, 3);
    println!("{} {}", x, z); // 1 3

    // match 的默认分支
    let y = 5;
    match y {
        1 => println!("one"),
        _ => println!("other"),
    }
}

// 函数参数不使用时，避免 unused variable 警告
fn process(_unused: i32, value: i32) -> i32 {
    value * 2
}

带下划线前缀的变量名（_name）告诉编译器"我知道这个变量没有用到，不要报警告"，但变量仍然会被绑定。而单独的 _ 则完全不绑定。

fn main() {
    let _x = 5;  // 不报 unused 警告，但 x 被绑定了
    let _ = 5;   // 完全不绑定，立即丢弃
}

@ 绑定：匹配的同时捕获值

@ 让你在测试一个值是否匹配模式的同时，把这个值绑定到一个变量名上：

fn main() {
    let num = 7;

    match num {
        // 测试是否在 1..=12 范围内，同时把值绑定到 n
        n @ 1..=12 => println!("月份：{}", n),
        n @ 13..=19 => println!("青少年：{}", n),
        _ => println!("其他"),
    }
}

没有 @ 时，范围模式只能判断是否匹配，无法在 arm 里用到原始值。@ 解决了这个问题：

fn classify_score(score: u32) -> String {
    match score {
        s @ 90..=100 => format!("优秀（{}分）", s),
        s @ 75..=89  => format!("良好（{}分）", s),
        s @ 60..=74  => format!("及格（{}分）", s),
        s            => format!("不及格（{}分）", s),
    }
}

fn main() {
    println!("{}", classify_score(95)); // 优秀（95分）
    println!("{}", classify_score(58)); // 不及格（58分）
}

小结

这一篇覆盖了 Rust 中最高频的语法核心：

  • 变量绑定默认不可变，需要可变时加 mut；shadowing 可以改变类型
  • 基础类型精确到位数，char 是 4 字节 Unicode 标量
  • 复合类型：tuple 适合临时多值，struct 适合命名字段，enum 可以携带不同类型的数据
  • 函数返回值是最后一个表达式，加分号就变语句，这是 Rust 特有的认知
  • match 穷尽匹配，支持范围、多值、guard、解构，远比 switch 强大
  • if let / while let 是单分支 match 的简洁写法
  • .. / _ / @ 是模式匹配里的三个利器
