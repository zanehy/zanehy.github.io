04 | Lua 数据类型与变量

发表于： 2026-03-08 分类于： OpenResty
字数： 3027 阅读：≈ 7分钟 浏览：

上一篇我们搞清楚了 OpenResty 的执行生命周期，知道了每个阶段能做什么事情。从这篇开始，我们进入第二阶段：Lua 语言基础。

很多人问：学 OpenResty 必须先精通 Lua 吗？答案是不必要。Lua 是一门很小的语言，核心语法十分钟就能扫完。但有几个地方如果不搞清楚，后面写代码会踩很多坑。这篇文章就专门讲这些基础但重要的东西。

Lua 只有六种数据类型

很多编程语言有几十种数据类型，而 Lua 只有六种：

类型说明
nil空值，表示"不存在"
boolean布尔值，true 或 false
number数字（整数和浮点数统一用这个）
string字符串
table表，Lua 最核心的数据结构
function函数也是一种值

这六种类型就是 Lua 的全部。简单是 Lua 的设计哲学，也是它能嵌入 OpenResty 这种高性能系统的原因之一。

用 type() 函数可以查看一个值的类型，返回的是类型名称字符串：

print(type(nil))        -- nil
print(type(true))       -- boolean
print(type(42))         -- number
print(type(3.14))       -- number
print(type("hello"))    -- string
print(type({}))         -- table
print(type(print))      -- function

记住这个函数，后面判断类型时会经常用到。

nil：不存在就是 nil

nil 在 Lua 里表示"什么都没有"。一个变量如果没有赋值，它的值就是 nil；把一个变量设置为 nil，相当于删除这个变量。

local x
print(x)        -- nil，未赋值的变量默认是 nil

local t = {a = 1, b = 2}
t.b = nil       -- 把 t.b 从 table 里删掉
print(t.b)      -- nil

在 OpenResty 里，nil 经常出现在"取不到值"的场景。比如从 Redis 查一个不存在的 key，返回的就是 nil：

local res, err = red:get("nonexistent_key")
if res == nil then
    ngx.log(ngx.ERR, "redis error: ", err)
    return
end
if res == ngx.null then
    -- key 不存在（注意：这里是 ngx.null，不是 nil）
    ngx.say("key not found")
end

注意上面的 ngx.null——这是 OpenResty 特有的概念，后面讲 Redis 时会详细说。

boolean：只有两个值，但判断逻辑不一样

Lua 的 boolean 类型只有 true 和 false。但 Lua 的"真假判断"规则和其他语言不同：

在 Lua 里，只有 false 和 nil 是假，其他所有值都是真。

包括数字 0、空字符串 ""，在 Lua 里都是真！

if 0 then
    print("0 是真")    -- 这行会打印！
end

if "" then
    print("空字符串是真")    -- 这行也会打印！
end

这和 JavaScript、Python 的行为完全不同，是 Lua 最常见的坑之一。

number：只有一种数字类型

Lua 里没有 int、float、double 之分，所有数字都是 number 类型。底层默认是双精度浮点数（64-bit double），对大多数场景完全够用。

local a = 10        -- 整数写法
local b = 3.14      -- 浮点数写法
local c = 1e6       -- 科学计数法，等于 1000000
local d = 0xff      -- 十六进制，等于 255

print(type(a))      -- number
print(type(b))      -- number
print(10 / 3)       -- 3.3333333333333
print(10 // 3)      -- 3（整除，Lua 5.3+ 支持）
print(10 % 3)       -- 1（取余）

在 OpenResty 里，做限流计数、时间戳计算时经常用到数字运算，掌握这些基本操作就够了。

string：不可变的字节序列

Lua 的字符串是不可变的（immutable）。一旦创建就不能修改，所有"修改"操作都是返回一个新字符串。

字符串有三种写法：

local s1 = "双引号字符串"
local s2 = '单引号字符串'
local s3 = [[
多行字符串
第二行
第三行
]]    -- 双方括号，适合写 HTML 片段或多行 JSON 模板

字符串可以用 .. 拼接，用 # 获取长度：

local name = "OpenResty"
local greeting = "Hello, " .. name .. "!"
print(greeting)         -- Hello, OpenResty!
print(#greeting)        -- 17

注意：# 返回的是字节数，不是字符数。如果字符串包含中文（UTF-8），每个汉字占 3 个字节：

local s = "你好"
print(#s)    -- 6，不是 2

在 OpenResty 里处理中文内容时，这一点很重要。

table：Lua 的瑞士军刀

table 是 Lua 里最重要的数据结构，没有之一。数组、字典、对象、模块——在 Lua 里全靠 table 实现。

用作数组（下标从 1 开始）：

local fruits = {"apple", "banana", "cherry"}
print(fruits[1])    -- apple（注意：Lua 数组下标从 1 开始，不是 0）
print(#fruits)      -- 3

-- 遍历数组
for i, v in ipairs(fruits) do
    print(i, v)
end
-- 输出：
-- 1  apple
-- 2  banana
-- 3  cherry

用作字典（key-value）：

local user = {
    name = "Alice",
    age = 30,
    is_admin = false,
}
print(user.name)        -- Alice
print(user["age"])      -- 30（两种访问方式等价）

-- 遍历字典
for k, v in pairs(user) do
    print(k, v)
end

混合用法：

local config = {
    host = "127.0.0.1",
    port = 6379,
    "extra1",    -- 数字下标 1
    "extra2",    -- 数字下标 2
}
print(config.host)    -- 127.0.0.1
print(config[1])      -- extra1

在 OpenResty 里，table 无处不在：解析出来的请求参数是 table，Redis 返回的结果集是 table，自己写的模块也是 table。

function：函数是一等公民

在 Lua 里，函数和数字、字符串一样，都是普通的值，可以赋值给变量、作为参数传递、从函数里返回。

-- 普通函数定义
local function add(a, b)
    return a + b
end

-- 等价写法（函数赋值给变量）
local add = function(a, b)
    return a + b
end

print(add(3, 4))    -- 7

Lua 函数支持多返回值：

local function divmod(a, b)
    return a // b, a % b    -- 同时返回商和余数
end

local q, r = divmod(10, 3)
print(q, r)    -- 3  1

这在 OpenResty 里非常常见。大多数 API 调用都返回两个值：结果和错误信息：

local res, err = ngx.location.capture("/api/user")
if not res then
    ngx.log(ngx.ERR, "capture failed: ", err)
    return
end

弱类型的坑：你必须知道的陷阱

Lua 是动态类型语言，变量没有类型，值才有类型。这带来了灵活性，也带来了一些坑。

坑一：字符串和数字的隐式转换

Lua 会在运算时自动把字符串转换成数字，也会把数字转换成字符串：

print("10" + 5)     -- 15（字符串自动转数字）
print(10 .. "")     -- "10"（数字自动转字符串）

但这个转换不是万能的：

print("hello" + 1)  -- 报错：attempt to perform arithmetic on a string value

建议：不要依赖隐式转换。需要转换时显式调用 tonumber() 和 tostring()：

local s = "42"
local n = tonumber(s)    -- 明确转换
if n then
    print(n + 1)    -- 43
end

坑二：== 不做类型转换

和 JavaScript 的 == 不同，Lua 的 == 不做类型转换：

print(1 == "1")     -- false（数字和字符串不相等）
print(0 == false)   -- false（数字和布尔值不相等）

坑三：table 比较的是引用 

local a = {1, 2, 3}
local b = {1, 2, 3}
local c = a

print(a == b)   -- false（不同的 table 对象）
print(a == c)   -- true（同一个 table 的引用）

要比较两个 table 的内容是否相同，需要自己写递归比较函数，或者用第三方库。

字符串操作：常用函数速查

Lua 的字符串函数都在 string 库里。在 OpenResty 里处理请求参数、URL、JSON 时经常用到。

string.len — 获取字节长度 

print(string.len("hello"))    -- 5
-- 等价于
print(("hello"):len())        -- 5（面向对象写法）
print(#"hello")               -- 5

string.sub — 截取子串

下标从 1 开始，支持负数（-1 表示最后一个字符）：

local s = "Hello, OpenResty!"
print(string.sub(s, 1, 5))     -- Hello
print(string.sub(s, 8))        -- OpenResty!
print(string.sub(s, -9))       -- OpenResty!（从倒数第9个开始）

string.find — 查找子串

返回起始和结束位置，找不到返回 nil：

local s = "content-type: application/json"
local start, finish = string.find(s, "application")
print(start, finish)    -- 16  26

-- 判断是否包含某个子串
if string.find(s, "json") then
    print("是 JSON 请求")
end

string.match — 模式匹配提取

比 find 更强大，支持捕获组：

local date = "2026-03-08"
local year, month, day = string.match(date, "(%d+)-(%d+)-(%d+)")
print(year, month, day)    -- 2026  03  08

-- 提取 URL 中的路径
local url = "/api/user/123"
local id = string.match(url, "/user/(%d+)")
print(id)    -- 123

string.gsub — 替换 

local s = "Hello World"
print(string.gsub(s, "World", "OpenResty"))    -- Hello OpenResty  1
-- 第二个返回值是替换次数

-- 用于清理多余空格
local messy = "  hello   world  "
print(string.gsub(messy, "%s+", " "))    -- hello world（简化版）

string.format — 格式化字符串

类似 C 的 printf，在拼接日志、构造响应时很有用：

local name = "Alice"
local age = 30
local s = string.format("用户 %s，年龄 %d", name, age)
print(s)    -- 用户 Alice，年龄 30

-- 在 OpenResty 里格式化 JSON 响应
local json_str = string.format('{"code":%d,"msg":"%s"}', 200, "ok")
ngx.say(json_str)

string.upper / string.lower — 大小写转换 

print(string.upper("hello"))    -- HELLO
print(string.lower("WORLD"))    -- world

-- 处理请求头时规范化大小写
local content_type = ngx.req.get_headers()["Content-Type"]
if content_type and string.lower(content_type):find("json") then
    -- 处理 JSON 请求
end

string.rep — 重复字符串 

print(string.rep("ab", 3))        -- ababab
print(string.rep("ab", 3, "-"))   -- ab-ab-ab（带分隔符，Lua 5.2+）

面向对象写法的语法糖

你可能注意到上面出现了 ("hello"):len() 这种写法。Lua 的字符串支持把 string.xxx(s, ...) 写成 s:xxx(...) 的形式，本质是一样的：

local s = "Hello, World"

-- 两种写法等价
string.upper(s)
s:upper()

-- 链式调用
local result = s:lower():gsub("world", "lua")
print(result)    -- hello, lua

在 OpenResty 代码里，你会同时看到这两种写法，都要能看懂。

小结

这篇文章覆盖了 Lua 的六种数据类型和常用字符串操作。几个关键点值得再强调：

  • 只有 nil 和 false 是假，数字 0 和空字符串都是真
  • # 返回字节数，不是字符数，处理中文时要注意
  • table 比较的是引用，不是内容
  • type() 函数是判断类型的标准方式
  • 显式转换优于隐式转换，tonumber() 和 tostring() 是你的朋友
