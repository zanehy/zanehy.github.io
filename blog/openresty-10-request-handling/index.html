10 | 请求处理：读取一切输入

发表于： 2026-03-15 分类于： OpenResty
字数： 1969 阅读：≈ 4分钟 浏览：

一个 HTTP 请求能携带的信息很多：URL 里的查询参数、POST 表单数据、各种请求头、JSON Body……OpenResty 提供了一套完整的 API 来读取这些输入。这篇文章逐一讲清楚每种输入的读取方式，以及各种坑。

读取查询参数

URL 里 ? 后面的部分就是查询参数，比如 /api/users?page=2&size=10&tag=lua&tag=openresty。

local args, err = ngx.req.get_uri_args()
if not args then
    ngx.log(ngx.ERR, "failed to get uri args: ", err)
    return
end

-- 读取单个参数
local page = args.page or "1"
local size = args["size"] or "10"

-- 参数有多个同名值时，是一个数组
local tags = args.tag
if type(tags) == "table" then
    -- tags = {"lua", "openresty"}
    for _, tag in ipairs(tags) do
        ngx.say("tag: ", tag)
    end
elseif type(tags) == "string" then
    ngx.say("tag: ", tags)
end

get_uri_args() 默认最多返回 100 个参数，超出的会被截断。如果你的接口参数可能很多，可以传入上限值：

local args = ngx.req.get_uri_args(200)   -- 最多返回 200 个
local args = ngx.req.get_uri_args(0)     -- 0 表示不限制（谨慎使用）

注意：所有参数值都是字符串，args.page 得到的是 "2" 而不是 2，需要用 tonumber() 转换。

读取 POST 参数

Content-Type: application/x-www-form-urlencoded 格式的 POST 请求：

-- 必须先读取 body！
ngx.req.read_body()

local args, err = ngx.req.get_post_args()
if not args then
    ngx.log(ngx.ERR, "failed to get post args: ", err)
    return
end

local username = args.username
local password = args.password

这里有个关键点：必须先调用 ngx.req.read_body()，否则 get_post_args() 读不到数据。

也可以在 nginx.conf 里配置让 Nginx 自动读取：

location /api/ {
    lua_need_request_body on;   -- 自动读取 body
    content_by_lua_block {
        local args = ngx.req.get_post_args()
        -- 不需要手动调用 read_body
    }
}

不过更推荐在代码里显式调用 read_body()，控制权更清晰。

读取请求头 

local headers = ngx.req.get_headers()

-- 常用请求头
local content_type = headers["content-type"]    -- 注意：键是小写
local authorization = headers["authorization"]
local user_agent = headers["user-agent"]
local x_forwarded_for = headers["x-forwarded-for"]

-- 获取客户端真实 IP（考虑代理）
local client_ip = headers["x-real-ip"] or headers["x-forwarded-for"] or ngx.var.remote_addr

get_headers() 返回的所有键都是小写的，与 HTTP 头原始大小写无关。

同一个头有多个值时（这在合法的 HTTP 里是允许的），返回的是数组：

local accept = headers["accept"]
if type(accept) == "table" then
    -- 多个 Accept 头
    for _, v in ipairs(accept) do
        ngx.say("accept: ", v)
    end
end

同样，get_headers() 默认最多返回 100 个头。

读取请求 Body

基本读取 

ngx.req.read_body()

-- 获取 body 数据
local body = ngx.req.get_body_data()

if body then
    ngx.say("body length: ", #body)
    ngx.say("body: ", body)
else
    -- body 被写到临时文件（超过 client_body_buffer_size 时）
    local file = ngx.req.get_body_file()
    if file then
        ngx.say("body in file: ", file)
        -- 需要自己读文件
    end
end

为什么 get_body_data() 可能返回 nil？

Nginx 对请求 body 有缓冲大小限制（client_body_buffer_size，默认 8KB 或 16KB）。超过这个大小，body 会被写到临时文件，这时 get_body_data() 返回 nil，需要用 get_body_file() 获取文件路径再读取。

如果你的接口需要处理较大的 body，可以调整配置：

location /upload/ {
    client_max_body_size 10m;          -- 允许最大 10MB
    client_body_buffer_size 1m;        -- 1MB 以内直接缓存在内存
    content_by_lua_block {
        ngx.req.read_body()
        local body = ngx.req.get_body_data()
    }
}

处理 JSON Body

这是 API 开发里最常见的场景：

content_by_lua_block {
    local cjson = require "cjson.safe"   -- 用 safe 版本，避免解析失败抛异常
    
    ngx.req.read_body()
    local body = ngx.req.get_body_data()
    
    if not body or body == "" then
        ngx.status = 400
        ngx.say('{"code":400,"message":"empty body"}')
        return ngx.exit(ngx.OK)
    end
    
    local data, err = cjson.decode(body)
    if not data then
        ngx.status = 400
        ngx.say('{"code":400,"message":"invalid json: ' .. (err or "unknown") .. '"}')
        return ngx.exit(ngx.OK)
    end
    
    -- 现在可以用 data 了
    local name = data.name
    local age = data.age
    
    ngx.header["Content-Type"] = "application/json"
    ngx.say(cjson.encode({
        code = 0,
        message = "received",
        name = name,
        age = age
    }))
}

为什么用 cjson.safe 而不是 cjson？

cjson 在解析失败时会抛出异常，需要用 pcall 捕获。cjson.safe 在解析失败时返回 nil, error_message，更符合 OpenResty 的错误处理风格。

修改请求

有时候需要在转发前修改请求，OpenResty 也提供了对应的 API：

修改请求 URI 

-- 重写 URI
ngx.req.set_uri("/new/path")

-- 带 jump 参数，触发 Nginx 内部跳转（相当于 rewrite ... last）
ngx.req.set_uri("/new/path", true)

修改查询参数 

-- 设置单个参数
ngx.req.set_uri_args({page = "2", size = "20"})

-- 直接设置 query string
ngx.req.set_uri_args("page=2&size=20")

修改请求头 

-- 设置请求头
ngx.req.set_header("X-User-Id", "12345")
ngx.req.set_header("Authorization", "Bearer " .. token)

-- 删除请求头
ngx.req.clear_header("X-Internal-Token")

这在代理场景里很常用：验证完 Token 之后，把用户信息注入到请求头里传给后端服务。

修改请求 Body 

ngx.req.read_body()

-- 替换 body 内容
ngx.req.set_body_data('{"modified": true}')

-- 更新 Content-Length
ngx.req.set_header("Content-Length", #'{"modified": true}')

一个完整的请求解析示例

把各种读取方式组合起来，写一个通用的请求信息收集函数：

local function parse_request()
    local result = {}
    
    -- 基础信息
    result.method = ngx.req.get_method()
    result.uri = ngx.var.uri
    result.query = ngx.var.query_string
    
    -- 查询参数
    result.args = ngx.req.get_uri_args()
    
    -- 请求头
    result.headers = ngx.req.get_headers()
    result.client_ip = result.headers["x-real-ip"] 
                    or result.headers["x-forwarded-for"]
                    or ngx.var.remote_addr
    
    -- Body（只有 POST/PUT/PATCH 才读）
    local method = result.method
    if method == "POST" or method == "PUT" or method == "PATCH" then
        ngx.req.read_body()
        result.body_raw = ngx.req.get_body_data()
        
        local ct = result.headers["content-type"] or ""
        if ct:find("application/json", 1, true) then
            local cjson = require "cjson.safe"
            result.body_json, result.body_err = cjson.decode(result.body_raw or "")
        elseif ct:find("application/x-www-form-urlencoded", 1, true) then
            result.body_form = ngx.req.get_post_args()
        end
    end
    
    return result
end

常见坑汇总

坑一：忘记调用 read_body

读取 POST 参数或 body 之前必须先调 ngx.req.read_body()，这是最常见的遗漏。

坑二：read_body 调用时机

read_body 只能在 rewrite_by_lua、access_by_lua、content_by_lua 这些阶段调用，不能在 header_filter_by_lua 或 body_filter_by_lua 里调用。

坑三：参数值类型

所有从 get_uri_args 和 get_post_args 拿到的值都是字符串（或字符串数组）。数字比较前记得 tonumber()，否则 args.page > 10 会做字符串比较，结果可能出乎意料。

坑四：同名参数的处理

?a=1&a=2 这种情况，args.a 是 {"1", "2"}，不是 "1" 或 "2"。代码里要判断 type(args.a)。

小结

OpenResty 读取请求输入的 API 覆盖了所有场景：

  • ngx.req.get_uri_args()：查询参数
  • ngx.req.get_post_args()：POST 表单数据（先调 read_body）
  • ngx.req.get_headers()：请求头（键统一小写）
  • ngx.req.read_body() + get_body_data()：请求 Body
  • cjson.safe.decode()：解析 JSON Body
