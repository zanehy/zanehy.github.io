09 | ngx. 核心 API 全览

发表于： 2026-03-13 分类于： OpenResty
字数： 1560 阅读：≈ 4分钟 浏览：

前两个阶段我们把 Lua 语言本身搞清楚了。从这篇开始进入第三阶段——OpenResty 核心 API。这些 API 是你和 Nginx 打交道的直接工具，几乎每个请求处理逻辑里都会用到。

输出响应

ngx.say 和 ngx.print

这是最基础的输出函数，区别只有一个：ngx.say 末尾自动加换行，ngx.print 不加。

content_by_lua_block {
    ngx.say("Hello, World!")         -- 输出 "Hello, World!\n"
    ngx.print("no newline here")     -- 输出 "no newline here"
    ngx.say("line 1", "line 2")     -- 可以传多个参数，依次输出
}

两个函数都接受多个参数，效果等同于拼接后输出。传 nil 会被忽略，传 table 会报错。

ngx.send_headers

默认情况下，第一次调用 ngx.say 或 ngx.print 时响应头会自动发送。如果需要提前发送头，可以显式调用：

ngx.status = 200
ngx.header["Content-Type"] = "application/json"
ngx.send_headers()
-- 之后可以分块输出 body
ngx.say('{"status": "ok"}')

ngx.flush

强制把缓冲区里的内容发送给客户端（流式响应时用）：

for i = 1, 5 do
    ngx.say("chunk " .. i)
    ngx.flush(true)   -- true 表示同步等待发送完成
    ngx.sleep(0.5)
end

控制响应状态

ngx.status 

ngx.status = 404
ngx.say("Not Found")

ngx.status 必须在发送响应头之前设置，否则不生效。

ngx.exit

立即结束请求处理，返回指定状态码：

-- 返回 403 并终止
ngx.exit(403)

-- 返回 200（正常结束）
ngx.exit(ngx.OK)

-- 常用状态码常量
-- ngx.OK          = 200
-- ngx.HTTP_MOVED_TEMPORARILY = 302
-- ngx.HTTP_BAD_REQUEST       = 400
-- ngx.HTTP_UNAUTHORIZED      = 401
-- ngx.HTTP_FORBIDDEN         = 403
-- ngx.HTTP_NOT_FOUND         = 404
-- ngx.HTTP_INTERNAL_SERVER_ERROR = 500

ngx.exit 会立即中断当前阶段的 Lua 代码，跳转到 Nginx 的错误处理流程。如果在 content_by_lua 阶段调用 ngx.exit(ngx.OK)，相当于正常结束响应。

操作请求头和响应头

读取请求头 

-- 读取单个请求头（转小写）
local content_type = ngx.req.get_headers()["Content-Type"]
local auth = ngx.req.get_headers()["authorization"]

-- 一次性获取所有请求头
local headers = ngx.req.get_headers()
for k, v in pairs(headers) do
    ngx.log(ngx.INFO, k, ": ", v)
end

get_headers() 返回的键都是小写的。同名头存在多个值时，值是一个数组。

设置响应头 

-- 设置单个响应头
ngx.header["Content-Type"] = "application/json; charset=utf-8"
ngx.header["X-Request-Id"] = "abc-123"

-- 设置多个同名响应头（传 table）
ngx.header["Set-Cookie"] = {"session=abc", "token=xyz; HttpOnly"}

-- 删除响应头
ngx.header["X-Powered-By"] = nil

响应头必须在发送 body 之前设置，发出去之后修改无效。

时间相关

获取当前时间 

-- Unix 时间戳（秒，整数）
local t = ngx.time()

-- Unix 时间戳（秒，带毫秒小数）
local t = ngx.now()        -- 例如 1710000000.123

-- 转换为可读格式
local http_date = ngx.http_time(ngx.time())  -- "Mon, 10 Mar 2026 02:00:00 GMT"
local cookie_date = ngx.cookie_time(ngx.time())

重要：ngx.time() 和 ngx.now() 返回的是请求开始时缓存的时间，不是实时时钟。在一次请求处理过程中，多次调用返回的值相同（除非调用了 ngx.sleep 或其他 yield 操作来刷新缓存）。

如果需要请求处理过程中的实时时间，用 ngx.update_time() 先刷新：

ngx.update_time()
local now = ngx.now()

sleep 

-- 非阻塞 sleep，单位秒（支持小数）
ngx.sleep(0.1)   -- sleep 100ms
ngx.sleep(1)     -- sleep 1s

ngx.sleep 会挂起当前协程，让出 CPU 给其他请求，不会阻塞 worker 进程。

日志

ngx.log 

ngx.log(ngx.ERR, "something went wrong: ", err)
ngx.log(ngx.WARN, "deprecated usage detected")
ngx.log(ngx.INFO, "request processed in ", elapsed, "ms")
ngx.log(ngx.DEBUG, "debug info: ", cjson.encode(data))

日志级别常量（从高到低）：

常量对应 Nginx 日志级别
ngx.STDERRstderr
ngx.EMERGemerg
ngx.ALERTalert
ngx.CRITcrit
ngx.ERRerror
ngx.WARNwarn
ngx.NOTICEnotice
ngx.INFOinfo
ngx.DEBUGdebug

日志输出到 Nginx 的 error_log 文件，受 nginx.conf 里 error_log 指令的级别控制。级别低于配置值的日志会被忽略（不影响性能，但也不输出）。

注意：ngx.log 不接受 nil，传 nil 会出现 “nil” 字符串。多个参数会直接拼接，不加分隔符。

URI 和路径 

-- 当前请求的 URI（不含 query string）
local uri = ngx.var.uri           -- "/api/users"

-- 完整请求路径（含 query string）
local request_uri = ngx.var.request_uri  -- "/api/users?page=1"

-- URI 编解码
local encoded = ngx.escape_uri("hello world/test")  -- "hello%20world%2Ftest"
local decoded = ngx.unescape_uri("hello%20world")   -- "hello world"

-- query string 解析
local args = ngx.req.get_uri_args()
local page = args.page or "1"

几个实用的工具函数

ngx.md5 / ngx.sha1_bin 

local hash = ngx.md5("hello")           -- "5d41402abc4b2a76b9719d911017c592"
local bin = ngx.md5_bin("hello")        -- 二进制，16字节
local sha1 = ngx.sha1_bin("hello")      -- SHA1 二进制，20字节

ngx.encode_base64 / ngx.decode_base64 

local encoded = ngx.encode_base64("hello world")   -- "aGVsbG8gd29ybGQ="
local decoded = ngx.decode_base64("aGVsbG8gd29ybGQ=")  -- "hello world"

ngx.quote_sql_str

对字符串进行 MySQL 风格的转义，防 SQL 注入（但建议用预编译语句，这个只是备用）：

local safe = ngx.quote_sql_str(user_input)
local sql = "SELECT * FROM users WHERE name = " .. safe

一个完整的示例

把上面这些 API 组合起来，写一个简单的 JSON 接口：

content_by_lua_block {
    local cjson = require "cjson"
    
    -- 设置响应头
    ngx.header["Content-Type"] = "application/json; charset=utf-8"
    ngx.header["X-Request-Id"] = ngx.md5(tostring(ngx.now()))
    
    -- 读取参数
    local args = ngx.req.get_uri_args()
    local name = args.name
    
    if not name then
        ngx.status = 400
        ngx.say(cjson.encode({
            code = 400,
            message = "missing required param: name"
        }))
        return ngx.exit(ngx.OK)
    end
    
    -- 记录日志
    ngx.log(ngx.INFO, "greeting request for: ", name)
    
    -- 返回响应
    ngx.say(cjson.encode({
        code = 0,
        message = "Hello, " .. name .. "!",
        timestamp = ngx.time()
    }))
}

小结

ngx.* API 是 OpenResty 开发的基础工具箱，本篇覆盖了最常用的部分：

  • ngx.say / ngx.print：输出响应体
  • ngx.status / ngx.exit：控制响应状态
  • ngx.header：操作响应头；ngx.req.get_headers()：读请求头
  • ngx.time() / ngx.now()：获取时间（注意是缓存值）
  • ngx.log：按级别记录日志
  • ngx.sleep：非阻塞等待
