02 | 环境搭建与目录结构

发表于： 2026-03-05 分类于： OpenResty
字数： 2327 阅读：≈ 5分钟 浏览：

上一篇我们搞清楚了 OpenResty 是什么。从这篇开始，就要真正动手了。

工欲善其事，必先利其器。这篇文章的目标很简单：在你的机器上把 OpenResty 跑起来，理解它的目录结构，然后写出第一段能运行的 Lua 代码。

两种安装方式

OpenResty 的安装主要有两条路：包管理器安装和源码编译。对于学习阶段来说，包管理器安装就够了，速度快、省心。源码编译适合生产环境需要定制模块的场景，这篇先不展开。

另外还有一种方式是 Docker，适合不想污染本机环境、或者系统不方便装软件的情况，后面单独讲。

用包管理器安装（推荐）

OpenResty 官方提供了预编译的包，支持主流 Linux 发行版。

Ubuntu / Debian 

# 安装依赖
sudo apt-get install -y lsb-release curl

# 添加 OpenResty 官方 APT 源
curl -fsSL https://openresty.org/package/pubkey.gpg | sudo gpg --dearmor -o /usr/share/keyrings/openresty.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/openresty.gpg] http://openresty.org/package/ubuntu $(lsb_release -sc) main" \
    | sudo tee /etc/apt/sources.list.d/openresty.list > /dev/null

# 更新并安装
sudo apt-get update
sudo apt-get install -y openresty

CentOS / RHEL 

# 添加 OpenResty YUM 源
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://openresty.org/package/centos/openresty.repo

# 安装
sudo yum install -y openresty

安装完成后，验证一下是否成功：

openresty -v

如果看到类似 nginx version: openresty/1.x.x.x 的输出，说明安装成功。

默认安装路径是 /usr/local/openresty/，后面会详细介绍这个目录的结构。

用 Docker 快速体验

如果你不想直接在系统上安装，Docker 是最省事的方式。

# 拉取官方镜像
docker pull openresty/openresty:latest

# 运行一个临时容器，映射 8080 端口
docker run --rm -p 8080:80 openresty/openresty:latest

然后在浏览器访问 http://localhost:8080，如果看到 OpenResty 的欢迎页，说明容器运行正常。

对于学习来说，用 Docker 的好处是可以随时重置环境，不用担心配置文件改乱了没法还原。后面的代码示例都会以本机安装为主，但 Docker 版本的操作逻辑是一样的，只是文件路径需要通过 -v 挂载进去。

目录结构详解

安装完成后，先来认识一下 OpenResty 的目录布局。这一步很重要，后面写配置、写 Lua 代码都需要知道文件放在哪里。

/usr/local/openresty/
├── bin/                # 可执行文件
│   └── openresty       # 主程序，本质上就是 nginx
├── nginx/              # Nginx 相关文件（核心目录）
│   ├── conf/           # 配置文件
│   │   ├── nginx.conf  # 主配置文件
│   │   └── ...
│   ├── html/           # 默认静态文件目录
│   │   └── index.html
│   └── logs/           # 日志目录
│       ├── access.log
│       └── error.log
├── lualib/             # Lua 标准库和第三方库
│   ├── ngx/            # ngx.* 相关库
│   ├── resty/          # lua-resty-* 系列库
│   └── ...
├── luajit/             # LuaJIT 运行时
│   ├── bin/
│   └── lib/
└── site/               # 自定义 Lua 代码推荐放这里
    └── lualib/

几个关键目录说明：

nginx/conf/ — 所有的配置文件都在这里。主配置文件是 nginx.conf，你可以在这里写 include 指令引入其他配置文件，实际开发中通常会按功能拆分多个 .conf 文件。

lualib/resty/ — OpenResty 附带的 lua-resty-* 系列库都在这里，包括操作 Redis、MySQL、HTTP 客户端等模块。用 require "resty.redis" 这样的方式引入。

luajit/ — 内置的 LuaJIT 运行时，一般不需要直接操作这个目录，但如果你想用 luarocks 安装第三方 Lua 包，需要注意要安装到 OpenResty 内置的 LuaJIT 环境里，而不是系统的 Lua 环境。

nginx/logs/ — 日志文件的默认位置。调试阶段 error.log 会是你最常查看的文件。

常用命令

在开始写代码之前，先熟悉一下日常会用到的几个命令：

# 启动
sudo openresty

# 停止
sudo openresty -s stop

# 优雅停止（等待当前请求处理完再退出）
sudo openresty -s quit

# 重新加载配置（不停服务，最常用）
sudo openresty -s reload

# 检查配置文件语法是否正确
sudo openresty -t

# 指定配置文件启动
sudo openresty -c /path/to/your/nginx.conf

openresty -t 这个命令非常重要。每次修改完配置文件，先用它检查一遍语法，确认没有错误再 reload，避免把服务搞挂。

第一个 Hello World

环境有了，目录认识了，接下来写第一段 Lua 代码。

修改配置文件

打开主配置文件：

sudo vim /usr/local/openresty/nginx/conf/nginx.conf

找到 server 块，在里面添加一个新的 location：

server {
    listen       80;
    server_name  localhost;

    # 新增这个 location
    location /hello {
        default_type text/plain;
        content_by_lua_block {
            ngx.say("Hello, OpenResty!")
        }
    }

    # 原有的 location / 保持不变
    location / {
        root   html;
        index  index.html index.htm;
    }
}

保存后，检查语法并重载：

sudo openresty -t
sudo openresty -s reload

然后测试：

curl http://localhost/hello

如果看到输出 Hello, OpenResty!，恭喜你，第一段 Lua 代码运行成功了。

代码解释

来看一下这几行配置的含义：

location /hello {
    default_type text/plain;
    content_by_lua_block {
        ngx.say("Hello, OpenResty!")
    }
}

default_type text/plain — 告诉浏览器返回的是纯文本内容。

content_by_lua_block { ... } — 这是 OpenResty 的关键指令，表示"在处理响应内容这个阶段，执行花括号里的 Lua 代码"。这个阶段对应的是 Nginx 请求处理的 content 阶段，下一篇会详细讲所有阶段。

ngx.say("Hello, OpenResty!") — ngx 是 OpenResty 提供的全局对象，包含了大量操作请求和响应的方法。ngx.say() 的作用是向客户端输出一行内容，并自动加上换行符。

稍微复杂一点

Hello World 太简单，再加点东西：

location /info {
    default_type application/json;
    content_by_lua_block {
        local cjson = require "cjson"
        
        local info = {
            message = "Hello from OpenResty",
            method  = ngx.req.get_method(),
            uri     = ngx.var.uri,
            time    = ngx.now()
        }
        
        ngx.say(cjson.encode(info))
    }
}

重载配置后访问：

curl http://localhost/info

你会看到类似这样的 JSON 输出：

{"message":"Hello from OpenResty","method":"GET","uri":"/info","time":1741234567.123}

这里用到了几个新东西：

  • require "cjson" — 引入 JSON 序列化库，OpenResty 内置了 cjson，直接 require 就能用
  • ngx.req.get_method() — 获取请求的 HTTP 方法（GET、POST 等）
  • ngx.var.uri — 读取 Nginx 内置变量，$uri 就是请求的路径
  • ngx.now() — 获取当前时间戳（精确到毫秒）

这几行代码展示了 OpenResty 最核心的编程模式：在 Lua 里读取请求信息、处理数据、输出响应。后续的一切都是在这个基础上扩展。

分离 Lua 代码到独立文件

把 Lua 代码全写在 nginx.conf 里，时间长了会很难维护。实际开发中，应该把 Lua 代码放到独立的 .lua 文件里，然后在配置里引用。

第一步：创建 Lua 文件目录和文件

sudo mkdir -p /usr/local/openresty/nginx/lua
sudo vim /usr/local/openresty/nginx/lua/hello.lua

写入以下内容：

local cjson = require "cjson"

local info = {
    message = "Hello from external Lua file",
    method  = ngx.req.get_method(),
    uri     = ngx.var.uri,
    time    = ngx.now()
}

ngx.say(cjson.encode(info))

第二步：在 nginx.conf 里告诉 OpenResty 去哪里找 Lua 文件

在 http 块里添加 lua_package_path：

http {
    # 告诉 OpenResty 到哪里找 .lua 文件
    lua_package_path "/usr/local/openresty/nginx/lua/?.lua;;";

    server {
        listen 80;

        location /info {
            default_type application/json;
            content_by_lua_file lua/hello.lua;
        }
    }
}

lua_package_path 里的 ? 是占位符，会被实际的模块名替换。;; 末尾的两个分号表示追加默认的搜索路径。

content_by_lua_file 和 content_by_lua_block 的区别是：前者从外部文件加载 Lua 代码，后者直接内联在配置里。文件方式更适合正式开发。

重载配置，访问 /info，效果和之前完全一样，只是代码现在在独立文件里了。

小结

这篇文章走完了 OpenResty 的最基础流程：

  • 在 Linux 上用包管理器安装 OpenResty
  • Docker 方式快速体验
  • 认识了安装目录的结构，知道配置、日志、Lua 库分别在哪里
  • 掌握了启动、停止、重载、检查语法这几个常用命令
  • 写出了第一段运行在 OpenResty 里的 Lua 代码
  • 学会了把 Lua 代码拆到独立文件里
