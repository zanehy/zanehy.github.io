04 | 高频语法精讲（下）——集合、字符串与错误速通

发表于： 2026-04-19 分类于： Rust
字数： 2350 阅读：≈ 5分钟 浏览：

上一篇把变量、类型和控制流讲完了。这一篇继续扫尾——把日常开发里用得最多的五个"基础设施"过一遍：两个集合类型、字符串的基本认知、以及错误处理的入门写法。

这些内容本身不复杂，但每一个都有值得单独说清楚的地方。

Vec：Rust 的动态数组

Vec<T> 是 Rust 里最常用的集合，对应 Go 的 slice、Python 的 list。

fn main() {
    // 创建
    let mut v: Vec<i32> = Vec::new();
    let v2 = vec![1, 2, 3]; // 宏语法，更常用

    // 添加元素
    v.push(10);
    v.push(20);
    v.push(30);

    // 访问：两种方式
    let first = v[0];        // 越界会 panic
    let second = v.get(1);   // 返回 Option<&i32>，更安全

    // 遍历
    for item in &v {
        println!("{}", item);
    }

    // 遍历并修改
    for item in &mut v {
        *item *= 2;
    }

    println!("{:?}", v); // [20, 40, 60]
}

几个常用操作：

let mut v = vec![3, 1, 4, 1, 5, 9, 2, 6];

v.sort();                    // 排序
v.dedup();                   // 去重（需要先排序）
v.retain(|&x| x > 3);       // 保留满足条件的元素
let sum: i32 = v.iter().sum(); // 求和
let len = v.len();           // 长度
v.pop();                     // 弹出最后一个
v.insert(0, 99);             // 在指定位置插入

和 Go slice 最大的区别：Vec 有明确的所有权，把 Vec 传给函数会发生 move，除非你传引用。这一点到所有权章节会详细讲，现在先有个印象。

HashMap：键值存储 

use std::collections::HashMap;

fn main() {
    let mut scores: HashMap<String, i32> = HashMap::new();

    // 插入
    scores.insert(String::from("Alice"), 100);
    scores.insert(String::from("Bob"), 85);

    // 读取
    let alice_score = scores.get("Alice"); // Option<&i32>

    // 遍历
    for (name, score) in &scores {
        println!("{}: {}", name, score);
    }

    // 不存在才插入（非常惯用）
    scores.entry(String::from("Carol")).or_insert(70);

    // 基于旧值更新
    let count = scores.entry(String::from("Alice")).or_insert(0);
    *count += 10; // Alice 变为 110
}

entry().or_insert() 是 Rust HashMap 最常见的写法，处理"统计词频"这类场景非常简洁：

use std::collections::HashMap;

fn word_count(text: &str) -> HashMap<&str, usize> {
    let mut map = HashMap::new();
    for word in text.split_whitespace() {
        let count = map.entry(word).or_insert(0);
        *count += 1;
    }
    map
}

String vs &str：先给结论

Rust 有两种字符串类型，这是初学者最容易困惑的地方之一。现在先给结论，后面所有权章节会解释为什么。

结论：

  • String：拥有所有权的字符串，存在堆上，可以修改，可以增长
  • &str：字符串切片，是对某段 UTF-8 字节序列的借用，不可修改
fn main() {
    let s1: &str = "hello world";    // 字符串字面量，类型是 &str
    let s2: String = String::from("hello world"); // 堆上的字符串
    let s3: String = "hello world".to_string();   // 同上

    // String 可以修改
    let mut s = String::from("hello");
    s.push_str(", world");
    s.push('!');
    println!("{}", s); // hello, world!

    // String 可以转成 &str（廉价操作）
    let slice: &str = &s;
    let slice2: &str = &s[0..5]; // "hello"
}

函数参数怎么写：

// 推荐：接受 &str，兼容性更好
fn greet(name: &str) {
    println!("Hello, {}!", name);
}

fn main() {
    let owned = String::from("Alice");
    greet(&owned);   // String 可以自动转 &str
    greet("Bob");    // &str 字面量直接传
}

函数参数优先用 &str，因为 String 可以自动转成 &str，反过来不行。

Option：处理可能不存在的值

Rust 没有 null。一个值可能不存在时，用 Option<T> 表达：

enum Option<T> {
    Some(T),
    None,
}

用法：

fn divide(a: f64, b: f64) -> Option<f64> {
    if b == 0.0 {
        None
    } else {
        Some(a / b)
    }
}

fn main() {
    match divide(10.0, 2.0) {
        Some(result) => println!("结果：{}", result),
        None => println!("除数不能为零"),
    }

    // 更简洁的写法
    let result = divide(10.0, 0.0);

    // unwrap_or：提供默认值
    let val = result.unwrap_or(0.0);

    // unwrap_or_else：用闭包计算默认值
    let val2 = result.unwrap_or_else(|| f64::INFINITY);

    // if let：只关心 Some 的情况
    if let Some(v) = divide(9.0, 3.0) {
        println!("得到 {}", v);
    }

    // map：对 Some 内部的值做变换
    let doubled = divide(4.0, 2.0).map(|x| x * 2.0);
    println!("{:?}", doubled); // Some(4.0)
}

常用方法速查：

let opt: Option<i32> = Some(42);

opt.unwrap()              // 取值，None 时 panic
opt.unwrap_or(0)          // 取值，None 时返回默认值
opt.unwrap_or_default()   // 取值，None 时返回类型默认值
opt.is_some()             // 是否有值
opt.is_none()             // 是否为空
opt.map(|x| x + 1)        // 变换内部值
opt.and_then(|x| ...)     // 链式操作（flatMap）
opt.filter(|x| *x > 10)  // 过滤

和 Go 的 value, ok 模式比：Rust 的 Option 强制你处理缺值情况，编译器不让你忘记。

Result：处理可能失败的操作

Result<T, E> 是 Rust 的错误处理核心：

enum Result<T, E> {
    Ok(T),
    Err(E),
}

基本用法：

use std::num::ParseIntError;

fn parse_age(s: &str) -> Result<u32, ParseIntError> {
    let age = s.parse::<u32>()?; // ? 运算符：遇到错误直接返回
    Ok(age)
}

fn main() {
    match parse_age("25") {
        Ok(age) => println!("年龄：{}", age),
        Err(e) => println!("解析失败：{}", e),
    }

    // unwrap_or、map、and_then 和 Option 一样可以用
    let age = parse_age("abc").unwrap_or(0);
    println!("年龄（容错）：{}", age);
}

? 运算符：错误传播的语法糖

? 是 Rust 里最值得立刻学会的语法糖。它的作用是：如果是 Err，立即 return；如果是 Ok，取出内部值继续执行。

use std::fs;
use std::io;

// 不用 ?，很啰嗦
fn read_file_verbose(path: &str) -> Result<String, io::Error> {
    let content = match fs::read_to_string(path) {
        Ok(s) => s,
        Err(e) => return Err(e),
    };
    Ok(content)
}

// 用 ?，简洁
fn read_file(path: &str) -> Result<String, io::Error> {
    let content = fs::read_to_string(path)?;
    Ok(content)
}

// 多个可能失败的操作连续用 ?
fn process_config(path: &str) -> Result<u64, io::Error> {
    let content = fs::read_to_string(path)?;
    let trimmed = content.trim();
    let value: u64 = trimmed.parse().map_err(|_| {
        io::Error::new(io::ErrorKind::InvalidData, "不是有效数字")
    })?;
    Ok(value)
}

? 只能用在返回 Result 或 Option 的函数里，在 main 函数里也可以用，只需要把签名改成：

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let content = std::fs::read_to_string("config.txt")?;
    println!("{}", content);
    Ok(())
}

几个不常用但核心的语法

最后补充三个在代码里经常出现、但初学时容易跳过的语法。

impl Trait：简化的泛型写法 

// 传统泛型写法
fn print_item<T: std::fmt::Display>(item: T) {
    println!("{}", item);
}

// impl Trait 写法（效果相同，更简洁）
fn print_item(item: impl std::fmt::Display) {
    println!("{}", item);
}

// 作为返回值（隐藏具体类型）
fn make_greeting(name: &str) -> impl std::fmt::Display {
    format!("Hello, {}!", name)
}

impl Trait 是编译期静态分发，和泛型本质相同，只是写起来更简洁。

type 别名 

type Kilometers = i32;
type Result<T> = std::result::Result<T, String>;

fn distance() -> Kilometers {
    42
}

// 在一个模块里统一定义错误类型后，可以简写所有函数签名
fn parse(s: &str) -> Result<i32> {
    s.parse::<i32>().map_err(|e| e.to_string())
}

type 别名不创建新类型，只是换个名字，两者完全可以互换。

const 和 static 

// const：编译期常量，可以内联到每个使用处
const MAX_CONNECTIONS: usize = 100;
const PI: f64 = 3.14159265358979;

// static：全局变量，有固定内存地址
static GREETING: &str = "Hello, World!";

// static mut：可变全局变量（需要 unsafe 访问，生产中避免使用）
static mut COUNTER: u32 = 0;

const 和 static 都是全局的，区别在于：const 每次使用都会内联（类似 C 的 #define），static 有固定地址（类似 C 的全局变量）。大多数场景用 const 就够了。

小结

这两篇语法精讲覆盖了 Rust 日常开发最高频的基础部分：

  • Vec 和 HashMap：掌握基本操作和 entry 模式
  • String vs &str：函数参数优先用 &str，String 可自动转换
  • Option 和 Result：Rust 的两大"容器"，取代 null 和异常
  • ? 运算符：错误传播的惯用写法，让代码干净利落
  • impl Trait / type / const：出镜率高的辅助语法

语法基础到这里打完了。从下一篇开始，进入 Rust 最核心也最有挑战性的部分——所有权系统。所有权是 Rust 与其他语言最根本的区别，也是读懂任何 Rust 代码的前提。
