15 | 连接 Redis：基础篇

发表于： 2026-03-23 分类于： OpenResty
字数： 1804 阅读：≈ 4分钟 浏览：

从这篇开始进入第四阶段：中间件与存储。Redis 是 OpenResty 项目里最常见的搭档，几乎所有需要缓存、计数、会话存储的场景都少不了它。这篇先把基础打牢：如何连接、常用命令怎么用、连接池怎么配置。

安装 lua-resty-redis

lua-resty-redis 是 OpenResty 官方提供的 Redis 客户端，基于 cosocket，完全非阻塞。OpenResty 发行版通常已经内置，不需要单独安装。

检查是否可用：

local ok, redis = pcall(require, "resty.redis")
if ok then
    print("lua-resty-redis available")
end

如果没有，可以用 OPM（OpenResty Package Manager）安装：

opm get openresty/lua-resty-redis

基本连接 

local redis = require "resty.redis"

local red = redis:new()

-- 设置超时（连接超时、读超时、发送超时，单位毫秒）
red:set_timeouts(1000, 1000, 1000)

-- 连接
local ok, err = red:connect("127.0.0.1", 6379)
if not ok then
    ngx.log(ngx.ERR, "failed to connect redis: ", err)
    return ngx.exit(500)
end

-- 如果 Redis 设置了密码
local ok, err = red:auth("your_password")
if not ok then
    ngx.log(ngx.ERR, "redis auth failed: ", err)
    return ngx.exit(500)
end

-- 选择数据库（默认是 0）
red:select(1)

常用命令

lua-resty-redis 的 API 设计很直接——Redis 命令名就是方法名，参数一一对应。

String 操作 

-- SET key value [EX seconds] [NX]
local ok, err = red:set("username", "alice")
local ok, err = red:set("session:abc", "user_data", "EX", 3600)  -- 带过期时间
local ok, err = red:set("lock:key", 1, "NX", "EX", 30)           -- NX：不存在才设置

-- GET
local val, err = red:get("username")
if val == ngx.null then
    -- key 不存在时返回 ngx.null，不是 nil
    val = nil
end

-- MGET 批量获取
local vals, err = red:mget("key1", "key2", "key3")
-- vals 是数组，不存在的 key 对应 ngx.null

-- INCR / INCRBY
local new_val, err = red:incr("counter")
local new_val, err = red:incrby("counter", 5)
local new_val, err = red:incrbyfloat("score", 1.5)

-- DEL
local count, err = red:del("key1", "key2")

-- EXISTS
local exists, err = red:exists("username")  -- 1 表示存在，0 表示不存在

-- EXPIRE / TTL
red:expire("session:abc", 1800)   -- 重置过期时间
local ttl, err = red:ttl("session:abc")  -- 剩余秒数，-1 永不过期，-2 不存在

Hash 操作 

-- HSET
red:hset("user:123", "name", "alice")
red:hset("user:123", "age", "28")
red:hmset("user:123", "name", "alice", "age", "28", "email", "alice@example.com")

-- HGET / HMGET
local name = red:hget("user:123", "name")
local vals = red:hmget("user:123", "name", "age", "email")

-- HGETALL（返回 field-value 交替的数组）
local data = red:hgetall("user:123")
-- data = {"name", "alice", "age", "28", "email", "alice@example.com"}

-- 转换为 table
local function hgetall_to_table(arr)
    if arr == ngx.null then return nil end
    local t = {}
    for i = 1, #arr, 2 do
        t[arr[i]] = arr[i + 1]
    end
    return t
end
local user = hgetall_to_table(data)
-- user = {name="alice", age="28", email="alice@example.com"}

-- HDEL / HEXISTS
red:hdel("user:123", "age")
local ok = red:hexists("user:123", "name")

List 操作 

-- LPUSH / RPUSH（左/右插入）
red:lpush("task_queue", "task1")
red:rpush("task_queue", "task2")

-- LPOP / RPOP（左/右弹出）
local task = red:lpop("task_queue")

-- LRANGE（获取范围，0 -1 获取全部）
local tasks = red:lrange("task_queue", 0, -1)

-- LLEN
local len = red:llen("task_queue")

Set 和 Sorted Set 

-- SET（集合）
red:sadd("tags", "lua", "openresty", "nginx")
local members = red:smembers("tags")
local is_member = red:sismember("tags", "lua")  -- 1 或 0

-- ZSET（有序集合）
red:zadd("leaderboard", 100, "alice")
red:zadd("leaderboard", 200, "bob")
red:zadd("leaderboard", 150, "charlie")

-- 按分数范围获取（从低到高）
local members = red:zrangebyscore("leaderboard", 0, 200)
-- 从高到低
local top3 = red:zrevrange("leaderboard", 0, 2, "WITHSCORES")

连接池配置

这是使用 lua-resty-redis 最重要的一个环节，每次用完连接后必须归还连接池，而不是直接关闭。

-- 错误做法：直接关闭连接
red:close()   -- 每次都销毁连接，下次重新建立，开销很大

-- 正确做法：放回连接池
local ok, err = red:set_keepalive(max_idle_timeout, pool_size)
-- max_idle_timeout：连接在池中最长空闲时间（毫秒）
-- pool_size：连接池大小（每个 worker 的连接数上限）

完整的使用模板：

local function get_redis()
    local redis = require "resty.redis"
    local red = redis:new()
    red:set_timeouts(1000, 1000, 1000)

    local ok, err = red:connect("127.0.0.1", 6379)
    if not ok then
        return nil, "connect failed: " .. (err or "")
    end
    return red
end

local function release_redis(red)
    -- 放回连接池：空闲超时 10 秒，池大小 100
    local ok, err = red:set_keepalive(10000, 100)
    if not ok then
        ngx.log(ngx.WARN, "failed to set keepalive: ", err)
        red:close()
    end
end

-- 使用
content_by_lua_block {
    local red, err = get_redis()
    if not red then
        ngx.log(ngx.ERR, err)
        return ngx.exit(500)
    end

    local val, err = red:get("some_key")

    release_redis(red)   -- 无论成功失败，都要归还

    if err then
        ngx.log(ngx.ERR, "redis get failed: ", err)
        return ngx.exit(500)
    end

    ngx.say(val ~= ngx.null and val or "not found")
}

连接池的参数设置

连接池大小的参考计算方式：

pool_size = (worker 可承受的最大并发数) / (单个请求的平均 Redis 操作耗时 / 请求平均耗时)

实际上，经验值是：每个 worker 配置 50～200 的连接池，根据业务压测调整。pool_size 不是越大越好——Redis 服务端也有最大连接数限制（maxclients），所有 worker 的连接数加起来不能超过这个值。

错误处理规范

OpenResty 的错误处理风格是返回值而不是异常。lua-resty-redis 的所有方法在失败时返回 nil, err_string：

local val, err = red:get("key")

-- 网络错误、命令执行错误
if err then
    ngx.log(ngx.ERR, "redis get error: ", err)
    release_redis(red)
    return ngx.exit(500)
end

-- key 不存在（不是错误，是正常情况）
if val == ngx.null then
    ngx.say("key not found")
    release_redis(red)
    return
end

-- 正常返回值
ngx.say(val)
release_redis(red)

特别注意 ngx.null 和 nil 的区别：

  • val == nil 表示 get 方法调用失败（err 不为空）
  • val == ngx.null 表示命令执行成功，但 key 在 Redis 里不存在

这两种情况处理方式完全不同，不能混淆。

一个封装好的 Redis 工具模块 

-- lib/redis_util.lua
local redis = require "resty.redis"
local ngx_log = ngx.log
local ngx_ERR = ngx.ERR

local _M = {}
local mt = { __index = _M }

local REDIS_HOST    = "127.0.0.1"
local REDIS_PORT    = 6379
local TIMEOUT_MS    = 1000
local KEEPALIVE_MS  = 10000
local POOL_SIZE     = 100

function _M.new()
    local red = redis:new()
    red:set_timeouts(TIMEOUT_MS, TIMEOUT_MS, TIMEOUT_MS)
    local ok, err = red:connect(REDIS_HOST, REDIS_PORT)
    if not ok then
        return nil, "redis connect failed: " .. err
    end
    return setmetatable({ red = red }, mt)
end

function _M:get(key)
    local val, err = self.red:get(key)
    if err then return nil, err end
    if val == ngx.null then return nil end
    return val
end

function _M:set(key, val, ttl)
    if ttl then
        return self.red:set(key, val, "EX", ttl)
    end
    return self.red:set(key, val)
end

function _M:del(key)
    return self.red:del(key)
end

function _M:close()
    local ok, err = self.red:set_keepalive(KEEPALIVE_MS, POOL_SIZE)
    if not ok then
        ngx_log(ngx_ERR, "redis keepalive failed: ", err)
        self.red:close()
    end
end

return _M

使用：

local redis_util = require "lib.redis_util"

local r, err = redis_util.new()
if not r then
    return ngx.exit(500)
end

local val = r:get("my_key")
r:set("another_key", "value", 300)
r:close()

小结

lua-resty-redis 的使用要掌握三件事：

  • 连接和基本命令操作——API 和 Redis 命令一一对应，上手很快
  • 连接池——用完连接必须 set_keepalive 归还，这是性能的关键
  • ngx.null 的判断——key 不存在时返回 ngx.null，不是 nil，不能混淆
